## Communication Patterns

### Direct Function Calls (Phase 1)
Agents are invoked directly by the Orchestrator via `agent.safe_handle()`.
No message queue needed. Simple, testable, no infrastructure dependency.

### Message Queue (Phase 2+, optional)
For scalability, agents could communicate via Redis pub/sub or RabbitMQ.
Each agent subscribes to relevant event topics.

## Data Flow

1. **Game moves** flow through the Rust engine (source of truth for board state)
2. **Coaching content** flows through the Python server (RAG + LLM pipeline)
3. **Player profiles** are persisted by MemoryAgent (JSON -> DB)
4. **LED commands** are generated by OutputAgent and sent via serial/GPIO
